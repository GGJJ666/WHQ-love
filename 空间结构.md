# 空间结构

## 概述

本文档描述了母AI Agent与子AI Agent之间的空间拓扑结构模型，旨在为多Agent系统的通信、调用与协作提供几何化的参考框架。

---

## 核心概念

### 母AI Agent：内切球

母AI Agent被抽象为空间中的一个**球体**，置于整个结构的中心位置。它承载系统的核心能力，包括：

- 基础模型（Shared Backbone）
- 通用技能模块
- 知识库与记忆接口
- 推理引擎

### 子AI Agent：正锥体顶点

子AI Agent被抽象为以母AI Agent球体为**内切球**的**正锥体**的各顶点。

| 子Agent数量 | 几何结构 | 说明 |
|:-----------:|:--------:|:-----|
| 4个 | 正三棱锥（正四面体） | 4个顶点，各对应一个子Agent |
| 6个 | 正八面体 | 6个顶点，各对应一个子Agent |
| 8个 | 正六面体（立方体） | 8个顶点，各对应一个子Agent |

> **默认结构**：4个子AI Agent → 正三棱锥（正四面体），球体为该正四面体的内切球。

---

## 信息传输机制

```
         子Agent A
        /    ↑    \
       ↓     |     ↑
  子Agent D  ●  子Agent B
       ↑     |     ↓
        \    ↓    /
         子Agent C
```

- **传输方向**：顺时针（Clockwise）
- **调用出入口**：正锥体各棱与球体的**切点**，共4条棱、4个切点，分别作为各子Agent与母Agent之间的调用接口

### 切点接口模型

```
          [子Agent A]
              |
         [切点 α]          ← 调用出口（子Agent A → 母Agent）
              |
         [母Agent ●]
              |
         [切点 β]          ← 调用入口（母Agent → 子Agent C）
              |
          [子Agent C]
```

每个切点代表一条信息通道：

| 切点 | 方向 | 描述 |
|:----:|:----:|:-----|
| α | 子→母 | 子Agent向母Agent提交任务/查询 |
| β | 母→子 | 母Agent向子Agent下发结果/指令 |

---

## 结构特性

1. **对称性**：正多面体保证每个子Agent与母Agent的距离相等，避免资源分配不均。
2. **可扩展性**：通过增加多面体阶数（正四面体→正八面体→…），可线性扩展子Agent数量。
3. **单点中心**：母Agent是唯一中心节点，所有子Agent通过切点接口与其通信，降低子Agent间的直接耦合。
4. **顺时针流转**：固定的信息流向减少冲突，便于调试与追踪。

## 与可行性论证的关联

本空间结构模型与[可行性论证（Issue #1）](https://github.com/GGJJ666/WHQ-love/issues/1)中的母—子多代理框架高度对应：

- 球体（母Agent） = 共享Backbone + 技能模块
- 锥体顶点（子Agent） = 轻量配置（LoRA/Adapter）+ 领域定制
- 切点接口 = 模块化调用接口（工具调用、技能注入）
